# Cursorist workshop

Materials for a hands-on workshop on building full-stack apps with **Cursor**: prompt craft, Cursor Rules, workflow patterns, and integrations (Prisma, Shadcn, Vercel-era tooling).

## What’s here

- **[prompts.md](./prompts.md)** — Exercise prompts in order: landing pages, architecture rules, task views with mock data, database migration with Prisma, bug fixes, Shadcn components, Plan Mode for new tasks, and optional **Google Stitch** design workflows.
- **[building-fullstack-apps-with-cursor.pdf](./building-fullstack-apps-with-cursor.pdf)** — Slides / reference for the “Building full-stack apps with Cursor” session.
- **`images/`** — Reference screenshots (e.g. Todoist-style landing, Cursorist landing, Today view).

The companion app you build in the workshop is a **to-do product named Cursorist** (today/inbox routes, due dates, checkboxes, seed data, and Prisma-backed persistence).

## Workshop application (code)

The **starter Next.js app** you update during the workshop lives in a separate repository:

- **[github.com/mauricedb/cursorist](https://github.com/mauricedb/cursorist)** — clone this and work against `main` (or the default branch) while following the prompts here.

For a **completed reference** after the exercises, use the same repo on the **`building-the-app`** branch.

## How to use this repo

1. Clone **[cursorist](https://github.com/mauricedb/cursorist)** and run installs / dev server from that project when prompts mention the app.
2. Open **Prompts.md** and work through the sections in sequence, or jump to the topic you need (rules, Prisma seed/studio, Stitch).
3. Use the PDF alongside for framing and extra context.
